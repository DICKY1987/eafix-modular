## Core Columns (9) - ALL Records

### 1. record_kind
- **Type**: `string` (enum)
- **Required**: Yes (all records)
- **Description**: Discriminator field that determines the record type
- **Valid Values**: `entity`, `edge`, `generator`
- **Example**: `"record_kind": "entity"`
- **Validation**: Must be one of the three enum values

### 2. record_id
- **Type**: `string`
- **Required**: Yes (all records)
- **Description**: Globally unique identifier for this registry record (primary key)
- **Format**: Typically `REC-NNNNNN` (6-digit counter)
- **Example**: `"record_id": "REC-000001"`
- **Validation**: Must be unique across all records in registry

### 3. status
- **Type**: `string` (enum)
- **Required**: Yes (all records)
- **Description**: Lifecycle status of the record
- **Valid Values**: 
  - `active` - Currently in use
  - `deprecated` - Marked for future removal
  - `quarantined` - Isolated due to issues
  - `archived` - Moved to long-term storage
  - `deleted` - Soft-deleted (marked for cleanup)
  - `closed` - Completed/finished (transient entities)
  - `running` - Currently executing (transient entities)
  - `pending` - Queued/waiting (transient entities)
  - `failed` - Execution failed (transient entities)
- **Example**: `"status": "active"`
- **Validation**: Must be one of the enum values
- **Notes**: First 5 values for files/assets/edges, last 4 for transient entities

### 4. notes
- **Type**: `string`
- **Required**: No (optional)
- **Description**: Free-text notes or comments about this record
- **Example**: `"notes": "Migrated from legacy system 2026-01-19"`
- **Validation**: None (free text)

### 5. tags
- **Type**: `array[string]`
- **Required**: No (optional)
- **Description**: Array of tag strings for categorization and search
- **Example**: `"tags": ["production", "critical", "v2.1"]`
- **Validation**: Must be array (can be empty)

### 6. created_utc
- **Type**: `string` (ISO 8601 datetime)
- **Required**: Yes (all records)
- **Description**: Timestamp when record was first created in registry
- **Format**: `YYYY-MM-DDTHH:MM:SS.ffffffZ`
- **Example**: `"created_utc": "2026-01-20T15:46:00.123456Z"`
- **Validation**: Must be valid ISO 8601 datetime with UTC timezone

### 7. updated_utc
- **Type**: `string` (ISO 8601 datetime)
- **Required**: Yes (all records)
- **Description**: Timestamp when record was last modified
- **Format**: `YYYY-MM-DDTHH:MM:SS.ffffffZ`
- **Example**: `"updated_utc": "2026-01-20T15:46:00.123456Z"`
- **Validation**: Must be valid ISO 8601 datetime; must be >= created_utc

### 8. created_by
- **Type**: `string`
- **Required**: Yes (all records)
- **Description**: User, tool, or system that created this record
- **Example**: `"created_by": "scanner"`, `"created_by": "admin@example.com"`
- **Validation**: Non-empty string

### 9. updated_by
- **Type**: `string`
- **Required**: Yes (all records)
- **Description**: User, tool, or system that last updated this record
- **Example**: `"updated_by": "migration_script"`, `"updated_by": "admin@example.com"`
- **Validation**: Non-empty string

## Entity Columns (41) - When record_kind = "entity"

### Identity (2 columns)

#### 10. entity_id
- **Type**: `string`
- **Required**: Yes (when record_kind = "entity")
- **Description**: Globally unique identifier for this entity (across all entity types)
- **Format**: Varies by entity_kind (16-digit ID for files, semantic ID for others)
- **Example**: `"entity_id": "0199900001260118"` (file), `"entity_id": "RUN-20260120-0001"` (transient)
- **Validation**: Must be unique across all entities

#### 11. entity_kind
- **Type**: `string` (enum)
- **Required**: Yes (when record_kind = "entity")
- **Description**: Discriminator for entity type (determines which fields are populated)
- **Valid Values**: `file`, `asset`, `transient`, `external`, `module`, `directory`, `process`, `other`
- **Example**: `"entity_kind": "file"`
- **Validation**: Must be one of enum values

---

### File-Specific Columns (10 columns) - When entity_kind = "file"

#### 12. doc_id
- **Type**: `string`
- **Required**: When entity_kind = "file"
- **Description**: 16-digit identity document ID for file
- **Format**: `NNNNNNNNNNNNNNNNN` (16 digits: namespace + type + scope + sequence)
- **Example**: `"doc_id": "0199900001260118"`
- **Validation**: 16-digit numeric string
- **Notes**: Typically same as entity_id for file entities

#### 13. filename
- **Type**: `string`
- **Required**: When entity_kind = "file"
- **Description**: Base name of the file (without directory path)
- **Example**: `"filename": "main.py"`, `"filename": "README.md"`
- **Validation**: Non-empty string

#### 14. extension
- **Type**: `string`
- **Required**: When entity_kind = "file"
- **Description**: File extension including the leading dot
- **Example**: `"extension": ".py"`, `"extension": ".md"`, `"extension": ""` (no extension)
- **Validation**: String (can be empty for files with no extension)
- **Derivation**: Extracted from filename

#### 15. relative_path
- **Type**: `string`
- **Required**: When entity_kind = "file"
- **Description**: Path to file relative to repository/project root
- **Format**: Forward slashes `/` as separator
- **Example**: `"relative_path": "src/core/main.py"`, `"relative_path": "README.md"`
- **Validation**: Non-empty string
- **Notes**: Use forward slashes even on Windows

#### 16. absolute_path
- **Type**: `string` or `null`
- **Required**: No
- **Description**: Full absolute filesystem path to file
- **Format**: Platform-specific (forward or back slashes)
- **Example**: `"absolute_path": "C:/projects/eafix/src/main.py"` or `null`
- **Validation**: Valid filesystem path or null
- **Notes**: Often null for portability; can be populated for system-specific operations

#### 17. directory_path
- **Type**: `string`
- **Required**: When entity_kind = "file"
- **Description**: Directory portion of relative_path (dirname)
- **Format**: Forward slashes `/` as separator
- **Example**: `"directory_path": "src/core"`, `"directory_path": ""` (root), `"directory_path": "."`
- **Validation**: String (empty string or "." for root-level files)
- **Derivation**: `os.path.dirname(relative_path)` using forward slashes

#### 18. size_bytes
- **Type**: `integer` or `null`
- **Required**: No
- **Description**: File size in bytes
- **Example**: `"size_bytes": 12345`, `"size_bytes": null`
- **Validation**: Non-negative integer or null

#### 19. mtime_utc
- **Type**: `string` (ISO 8601 datetime) or `null`
- **Required**: No
- **Description**: File modification timestamp (last write time)
- **Format**: `YYYY-MM-DDTHH:MM:SS.ffffffZ`
- **Example**: `"mtime_utc": "2026-01-20T15:46:00.123456Z"`, `"mtime_utc": null`
- **Validation**: Valid ISO 8601 datetime or null

#### 20. sha256
- **Type**: `string` or `null`
- **Required**: No
- **Description**: SHA-256 hash of file contents (for integrity/deduplication)
- **Format**: 64-character hex string
- **Example**: `"sha256": "abc123def456..."`, `"sha256": null`
- **Validation**: 64-character hex string or null

#### 21. content_type
- **Type**: `string` or `null`
- **Required**: No
- **Description**: MIME type or content type classification
- **Example**: `"content_type": "text/markdown"`, `"content_type": "application/python"`
- **Validation**: Valid MIME type string or null

## Cross-Field Validation Rules

### Entity Records
- When `entity_kind = "file"`: columns 12-21 (file-specific) should be populated
- When `entity_kind = "asset"`: columns 22-25 (asset-specific) should be populated
- When `entity_kind = "transient"`: columns 26-29 (transient-specific) should be populated
- When `entity_kind = "external"`: columns 30-32 (external-specific) should be populated
- `updated_utc >= created_utc`
- `last_seen_utc >= first_seen_utc` (when both present)

### Edge Records
- `source_entity_id` and `target_entity_id` must reference existing entities
- `confidence` (when present) must be 0.0 ≤ confidence ≤ 1.0
- `rel_type` must be uppercase (normalize at ingestion)

### Generator Records
- `output_path` and `output_path_pattern` are mutually exclusive (use one or neither, not both)
- `declared_dependencies` values must be valid column names
- `sort_keys` values must be valid column names
